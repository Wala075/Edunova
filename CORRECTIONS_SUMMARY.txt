┌────────────────────────────────────────────────────────────────────────────────┐
│ 🔴 BUG #1: URISyntaxException                                                  │
└────────────────────────────────────────────────────────────────────────────────┘

  Erreur:
  ├─ java.net.URISyntaxException: Illegal character in query at index 203
  └─ Cause: Espaces non encodés dans l'URL OAuth2

  Avant:
  ├─ scope=openid email profile  ❌ (espaces non encodés)
  └─ login_hint=user@email.com   ❌ (caractères spéciaux non encodés)

  Après:
  ├─ scope=openid%20email%20profile  ✅ (espaces encodés)
  └─ login_hint=user%40email.com     ✅ (caractères spéciaux encodés)

  Solution:
  ├─ Utiliser URLEncoder.encode() pour tous les paramètres
  ├─ Encoder les paramètres avant de les ajouter à l'URL
  └─ Vérifier que l'URL est correctement formée

  Fichier Modifié:
  └─ GoogleOAuth2WindowController.java
     └─ Méthode: effectuerConnexionGoogle()

┌────────────────────────────────────────────────────────────────────────────────┐
│ 🔴 BUG #2: BindException                                                       │
└────────────────────────────────────────────────────────────────────────────────┘

  Erreur:
  ├─ java.net.BindException: Address already in use: bind
  └─ Cause: Port 8888 déjà utilisé par une autre instance

  Avant:
  ├─ HttpServer.create(new InetSocketAddress(8888), 0)  ❌ (port fixe)
  └─ Pas de gestion d'erreur si le port est occupé

  Après:
  ├─ Essayer le port 8888 en premier
  ├─ Si occupé, utiliser un port dynamique (0)
  └─ Récupérer le port réel utilisé

  Solution:
  ├─ Implémenter un try-catch pour gérer l'exception
  ├─ Utiliser un port dynamique en fallback
  ├─ Récupérer le port réel: server.getAddress().getPort()
  └─ Logger le port utilisé pour le débogage

  Fichier Modifié:
  └─ GoogleOAuth2WindowController.java
     └─ Méthode: demarrerServeurLocal()
